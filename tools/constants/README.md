# constants.py — named mathematical constant digit generator

Generates digit strings for named mathematical constants, formatted for
human readability and archival storage.

Generated files conform to the **Crowsong Archivist Header Format (CAHF)**
as specified in `draft-darley-crowsong-archivist-00`.  Any CAHF-conformant
tool (`archivist.py`, `texts.py`, `udhr.py`, etc.) can verify the output of
this tool, and this tool can verify any CAHF-stamped file that carries
`CONSTANT` and `DIGITS` fields.

Suitable for use as IV sources in the Channel Camouflage Layer and Mnemonic
Share Wrapping constructions.  Also useful as standalone printed reference
material.

> **Named constants are public deterministic reference material.**  They are
> suitable as declared IV anchors but provide **no secrecy, unpredictability,
> or cryptographic strength**.

Requires `mpmath` at generation time.  Generated files are plain UTF-8 text
with no runtime dependencies.

---

## Usage

```bash
python constants.py list
python constants.py digits <name> <count>
python constants.py show <name> <count>
python constants.py generate <name> <count> <outfile>
python constants.py verify <name> <infile>
```

## Examples

```bash
# List available constants
python constants.py list

# Print 100 digits of pi, plain (pipeable)
python constants.py digits pi 100

# Print 200 digits of phi, formatted for reading
python constants.py show phi 200

# Generate an archival CAHF-stamped file of 10,000 digits of e
python constants.py generate e 10000 docs/constants/e-10000.txt

# Verify a previously generated file
python constants.py verify pi docs/constants/pi-10000.txt
```

## Generated file format

Generated files are CAHF-stamped self-describing plain text, readable without
software and suitable for inclusion in a Vesper archive or any system that
implements `draft-darley-crowsong-archivist-00`.

```
# ARCHIVIST  v1.0
#
# TITLE      π — Pi
# CONSTANT   pi
# OEIS       A000796
# NOTE       Ratio of circumference to diameter. Infinite, non-repeating.
#
# DIGITS     10000
# FORMAT     decimal, 10 digits per group, 6 groups per line
#
# NOTE       Named constants are public deterministic reference material.
# NOTE       They provide no secrecy, unpredictability, or cryptographic
# NOTE       strength.  Suitable as declared IV anchors only.
#
# DATE       2026-04-10
# TOOL       tools/constants.py (Proper Tools SRL)
#
# CRC32      <8-character lowercase hex>
# SHA256     <64-character lowercase hex>
# ---END-HEADER---
3141592653 5897932384 6264338327 9502884197 1693993751 0582097494
4592307816 4062862089 9862803482 5342117067 9821480865 1328230664
...
```

**SHA256 and CRC32 are computed over the normalised formatted body** (spaces
and newlines included), not over the raw digit string.  Any CAHF-conformant
verifier can reproduce the check by extracting the body after the sentinel
line and applying the standard CAHF hashing contract (§4.4).

Both `SHA256` and `CRC32` are **mandatory** header fields.  Files lacking
either field will fail verification.  To upgrade a file generated by an older
tool, re-generate it with `constants.py generate`.

## As a library

```python
from constants import get_digits, format_digits, get_constant_meta

# Raw digits
digits = get_digits("pi", 1000)        # "31415926535897..."

# Formatted for display
print(format_digits(digits))

# Registry metadata
key, meta = get_constant_meta("phi")
print(meta["symbol"], meta["oeis"])   # φ  A001622
```

## Generating the docs/constants/ reference files

```bash
for name in pi e phi sqrt2 sqrt3 sqrt5 ln2 apery; do
    python constants.py generate $name 10000 docs/constants/${name}-10000.txt
done
```

Pre-generated 10,000-digit files for all eight constants are committed
to `docs/constants/` and verified against their declared SHA256 and CRC32
values.  To regenerate and verify:

```bash
for name in pi e phi sqrt2 sqrt3 sqrt5 ln2 apery; do
    python constants.py verify $name docs/constants/${name}-10000.txt
done
```

## Running the tests

```bash
# Using pytest (recommended)
pip install pytest
python -m pytest test_constants.py -v

# Using stdlib unittest (no dependencies)
python -m unittest test_constants -v
# or

```

The test suite covers:

- Canonical digit vectors against known-good expected outputs
- `format_digits` layout and line-wrapping rules
- CAHF body normalisation and the stamp/verify hash symmetry contract
- Generate/verify round-trip for all eight constants
- CRLF-encoded file round-trip
- Parser structural validation: missing marker, wrong version, missing sentinel
- Strict body character validation (junk injection)
- Duplicate singleton field rejection (SHA256, CRC32, CONSTANT, DIGITS)
- NOTE field accumulation as a list (repeatable field)
- Legacy colon-separator field parsing (CAHF §7.3)
- All verification failure modes: tampered body, wrong declared hash, missing fields
- `positive_int` argparse type
- CLI smoke tests including no-subcommand exit behaviour

## CAHF conformance notes

This tool implements CAHF stamping and verification locally, per the
independence requirement of `draft-darley-crowsong-archivist-00` §8.3.  It
does not import `archivist.py`.

Key implementation decisions:

- `SHA256` and `CRC32` are computed over the **normalised formatted body**,
  not the raw digit string.  This is required for cross-tool verifiability.
- `CRC32` is a **mandatory** field in files produced by this tool.  The CAHF
  spec does not define CRC32 as a standard field; this is a tool-specific
  extension for quick manual/transcription error checking.
- Singleton fields (`SHA256`, `CRC32`, `CONSTANT`, `DIGITS`, etc.) are
  rejected on duplication — a duplicate integrity field signals a malformed
  or tampered header.
- `NOTE` is the only repeatable field; multiple `NOTE` lines accumulate as a
  list.
- The ARCHIVIST marker is checked by exact equality against `"# ARCHIVIST
  v1.0"`, so version mismatches are caught explicitly.
- Legacy colon-separator fields (CAHF §7.3) are accepted on read but never
  emitted on write.

## Connection to the Crowsong stack

Named constants serve as public, indestructible, offline-reproducible
initialisation vectors for:

- **Channel Camouflage Layer** — transformation schedule derived from
  digits of a named constant at a declared offset
- **Mnemonic Share Wrapping** — IV anchor for the 3/3 construction;
  the constant is the public share, specified by name not value

Declared in the resource fork header as:

```
IV: PI · OFFSET/1000 · BASE/10
```

See `docs/mnemonic-shamir-sketch.md` for the full construction.
